## 1. 项目概述

### 1.1 项目背景

在量化金融行业的招聘过程中，HR需要处理大量简历，手工筛选效率低且容易遗漏优秀候选人。本项目旨在构建一个智能化的简历筛选Agent，能够：

1. **自动化评分**：对简历进行多维度量化评分
2. **智能初筛**：基于阈值自动筛选合格候选人
3. **个性化出题**：为通过初筛的候选人生成针对性面试题目

### 1.2 核心价值

- **提高效率**：从手工筛选（5-10分钟/份）降低到自动筛选（30秒/份）
- **标准化**：统一评分标准，减少人为主观性
- **智能化**：根据候选人背景生成个性化面试题目
- **成本优化**：未通过初筛的简历不生成题目，节省API成本

### 1.3 设计目标

#### 功能目标
- ✅ 支持PDF格式简历自动解析
- ✅ 多维度评分（项目、实习、技术栈、科研、竞赛）
- ✅ 固定阈值初筛（可配置）
- ✅ 分层面试题目生成（基础题+进阶题）
- ✅ 结构化输出（便于系统集成）

#### 非功能目标
- ✅ **高可靠性**：数据验证确保评分准确性
- ✅ **易扩展**：模块化设计支持功能扩展
- ✅ **易集成**：作为子Agent可被其他系统调用
- ✅ **成本可控**：条件执行避免不必要的API调用

### 1.4 技术选型

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| **Pydantic AI** | Agent框架 | 强类型结构化输出，确保数据质量 |
| **LangGraph** | 工作流编排 | 状态机模式，支持条件分支 |
| **LangChain** | 文档处理 | 成熟的PDF解析能力 |
| **OpenAI GPT-4** | LLM | 强大的中文理解和推理能力 |
| **Pydantic** | 数据验证 | 类型安全，自动验证 |

### 2.2 分层设计

#### Layer 1: 接口层 (Interface Layer)
- **文件：** `src/__init__.py`
- **职责：** 提供统一的对外接口
- **组件：** `ResumeScreeningAgent` 类
- **功能：** 
  - 初始化配置和工作流
  - 提供同步/异步API
  - 处理用户请求和返回结果

#### Layer 2: 编排层 (Orchestration Layer)
- **文件：** `src/workflow.py`
- **职责：** 协调各个处理节点
- **组件：** `create_resume_screening_workflow()`
- **功能：**
  - 定义工作流DAG
  - 实现条件分支逻辑
  - 管理状态传递

#### Layer 3: 处理层 (Processing Layer)
- **文件：** `src/nodes/*.py`
- **职责：** 执行具体处理任务
- **组件：** 
  - `parser.py` - PDF解析
  - `evaluator.py` - 评分处理
  - `checker.py` - 阈值检查
  - `generator.py` - 题目生成
- **功能：** 状态转换和错误处理

#### Layer 4: Agent层 (Agent Layer)
- **文件：** `src/agents/*.py`
- **职责：** LLM推理和结构化输出
- **组件：**
  - `evaluator.py` - 评分Agent
  - `question_generator.py` - 题目生成Agent
- **功能：** 调用GPT-4进行智能分析

#### Layer 5: 数据层 (Data Layer)
- **文件：** `src/models.py`, `src/config.py`
- **职责：** 数据模型和配置管理
- **组件：**
  - Pydantic数据模型（6个）
  - 配置管理
- **功能：** 数据验证和类型安全

### 2.3 工作流设计

```
┌─────────┐
│  START  │
└────┬────┘
     │
     ▼
┌─────────────────┐
│  parse_resume   │ ← LangChain PyPDFLoader
│  (PDF → Text)   │
└────┬────────────┘
     │
     ▼
┌──────────────────┐
│ evaluate_resume  │ ← Pydantic AI Evaluator Agent
│  (Text → Score)  │
└────┬─────────────┘
     │
     ▼
┌──────────────────┐
│ check_threshold  │ ← Python Logic
│ (Score → Pass?)  │
└────┬─────────────┘
     │
     ▼
   ┌───┐
   │IF?│ passed_screening == True?
   └─┬─┘
     │
  ┌──┴────────────┐
  │ YES           │ NO
  ▼               ▼
┌──────────────┐  ┌─────┐
│generate_     │  │ END │
│questions     │  └─────┘
│(Score+Text → │
│ Questions)   │ ← Pydantic AI Question Generator
└──┬───────────┘
   │
   ▼
┌─────┐
│ END │
└─────┘
```

**关键设计点：**

1. **条件执行**：check_threshold后的条件分支，未通过不生成题目，节省成本
2. **状态传递**：每个节点返回完整state，包含所有上游结果
3. **错误处理**：每个节点都有try-catch，错误记录在state["error"]中
4. **幂等性**：节点可以被多次调用而不产生副作用

## 3. 评分体系设计

### 3.1 评分维度

#### 基础分（100分）

| 维度 | 满分 | 权重 | 评分要点 |
|------|------|------|----------|
| **项目经历** | 30分 | 30% | 量化相关性、技术难度、角色贡献、成果影响 |
| **实习经历** | 25分 | 25% | 公司匹配度、岗位相关性、时长深度、工作成果 |
| **技术栈** | 25分 | 25% | 编程语言、量化工具、机器学习、数据库中间件 |
| **科研经历** | 20分 | 20% | 论文发表、研究质量、领域相关性、应用价值 |

#### 竞赛加分（15分）

独立加分项，不计入权重：
- 数学建模（美赛、国赛）
- 量化比赛（宽客、WorldQuant）
- 编程竞赛（ACM、LeetCode）
- 金融科技比赛

**总分范围：0-115分**

### 3.2 阈值机制

- **默认阈值：** 70分
- **可配置：** 支持每次调用自定义阈值
- **判断逻辑：** `final_score >= threshold` → 通过初筛

### 3.3 评分原则

1. **客观性**：基于简历事实，不做主观臆断
2. **相关性优先**：量化岗位相关经历权重更高
3. **深度优先**：一个深入项目 > 多个浅显项目
4. **理由透明**：每个维度必须给出详细评分理由
5. **严格边界**：分数不能超过各维度上限

## 4. 面试题目生成策略

### 4.1 分层出题模式

#### 基础题（2题）
- **目的：** 考察基本理解和项目概况
- **特点：** 覆盖面广、难度适中、相对开放
- **示例：**
  - "请介绍你最擅长的量化策略项目"
  - "你在项目中使用了哪些技术栈？为什么选择这些技术？"

#### 进阶题（2-3题）
- **目的：** 深入考察技术能力和问题解决能力
- **特点：** 针对性强、有技术深度、基于具体项目
- **示例（针对CTA策略）：**
  - "在你的CTA策略中，如何处理信号延迟和滑点的影响？"
  - "你的回测系统如何保证历史数据的准确性和完整性？"

### 4.2 出题原则

1. **基于简历**：题目必须与候选人简历内容相关
2. **避免超纲**：不问简历中未涉及的领域
3. **有深度**：能考察真实能力，不是简单概念题
4. **可回答性**：候选人能够回答，避免过于刁钻
5. **多样性**：覆盖不同方面（策略、系统、数据、风控）

### 4.3 题目结构

每个题目包含：
- `level`: "basic" 或 "advanced"
- `question`: 题目内容
- `related_project`: 关联的项目名称
- `focus_area`: 考察重点（如"策略设计"、"系统架构"）

## 5. 数据流设计

### 5.1 输入数据

```python
{
    "resume_path": "path/to/resume.pdf",  # PDF文件路径
    "threshold": 70.0                      # 可选：自定义阈值
}
```

### 5.2 中间状态

```python
AgentState = {
    "resume_path": str,                    # 原始输入
    "resume_text": str,                    # 解析后文本
    "evaluation": EvaluationResult,        # 评分结果
    "interview_questions": InterviewQuestions,  # 面试题目（可选）
    "threshold": float,                    # 阈值
    "should_generate_questions": bool,     # 是否生成题目
    "error": Optional[str]                 # 错误信息
}
```

### 5.3 输出数据

```python
{
    "evaluation": {
        "candidate_name": "张三",
        "base_score": {...},               # 基础评分详情
        "bonus_score": {...},              # 竞赛加分详情
        "final_score": 85.0,               # 总分
        "passed_screening": True,          # 是否通过
        "timestamp": "2026-06-15T10:30:00"
    },
    "interview_questions": {               # 仅当通过初筛时存在
        "candidate_name": "张三",
        "basic_questions": [...],          # 2个基础题
        "advanced_questions": [...],       # 2-3个进阶题
        "generation_timestamp": "..."
    },
    "error": None                          # 错误信息（如有）
}
```
